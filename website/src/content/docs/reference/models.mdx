## Image

Following is the structure of the `KelletubeImageObject`:

| Property | Type            |
| -------- | --------------- |
| width    | `int` or `null` |
| height   | `int` or `null` |
| url      | `string`        |

## User

Structure of the `KelletubeUserObject`, which is used to represent a user in Kelletube returned by Kelletube Plugins.

| Property    | Type                                               |
| ----------- | -------------------------------------------------- |
| id          | `string`                                           |
| name        | `string`                                           |
| externalUri | `string`                                           |
| images      | List of [`KelletubeImageObject`][KelletubeImageObject] |

> `externalUri` is a URL that points to the user's profile on the external service (e.g. Listenbrainz)

## Album

### KelletubeSimpleAlbumObject

Following is the structure of the `KelletubeAlbumObject`:

| Property    | Type                                                             |
| ----------- | ---------------------------------------------------------------- |
| id          | `string`                                                         |
| name        | `string`                                                         |
| externalUri | `string`                                                         |
| images      | List of [`KelletubeImageObject`][KelletubeImageObject]               |
| albumType   | `album`, `single` or `compilation`                               |
| artists     | List of [`KelletubeSimpleArtistObject`][KelletubeSimpleArtistObject] |
| releaseDate | `string` (YYYY-MM-DD format) or `null`                           |

### KelletubeFullAlbumObject

Following is the structure of the `KelletubeFullAlbumObject`:

| Property    | Type                                                             |
| ----------- | ---------------------------------------------------------------- |
| id          | `string`                                                         |
| name        | `string`                                                         |
| externalUri | `string`                                                         |
| images      | List of [`KelletubeImageObject`][KelletubeImageObject]               |
| albumType   | `album`, `single` or `compilation`                               |
| artists     | List of [`KelletubeSimpleArtistObject`][KelletubeSimpleArtistObject] |
| releaseDate | `string` (YYYY-MM-DD format)                                     |
| totalTracks | `number`                                                         |
| recordLabel | `string` or `null`                                               |

## Track

Following is the structure of the `KelletubeFullTrackObject`:

| Property                     | Type                                                             |
| ---------------------------- | ---------------------------------------------------------------- |
| id                           | `string`                                                         |
| name                         | `string`                                                         |
| externalUri                  | `string`                                                         |
| artists                      | List of [`KelletubeSimpleArtistObject`][KelletubeSimpleArtistObject] |
| album                        | [`KelletubeSimpleAlbumObject`][KelletubeSimpleAlbumObject]           |
| durationMs (in milliseconds) | `number`                                                         |
| explicit                     | `boolean`                                                        |
| [isrc][isrc_wiki]            | `string`                                                         |

> `isrc` stands for International Standard Recording Code, which is a unique identifier for tracks.
> It is used to identify recordings and is often used in music distribution and royalty collection. The format is typically a 12-character alphanumeric code.

## Search Response

Following is the structure of the `KelletubeSearchResponseObject`:

| Property  | Type                                                                 |
| --------- | -------------------------------------------------------------------- |
| albums    | List of [`KelletubeSimpleAlbumObject`][KelletubeSimpleAlbumObject]       |
| artists   | List of [`KelletubeFullArtistObject`][KelletubeFullArtistObject]         |
| playlists | List of [`KelletubeSimplePlaylistObject`][KelletubeSimplePlaylistObject] |
| tracks    | List of [`KelletubeFullTrackObject`][KelletubeFullTrackObject]           |

## Browse Section

Following is the structure of `KelletubeBrowseSectionObject`:

| Property    | Type             |
| ----------- | ---------------- |
| id          | `string`         |
| title       | `string`         |
| externalUri | `string`         |
| browseMore  | `boolean`        |
| items       | List of `Object` |

The `items` property array can contain multiple type of `Object` in it but it will always be

- [`KelletubeFullPlaylistObject`][KelletubeFullPlaylistObject]
- [`KelletubeFullAlbumObject`][KelletubeFullAlbumObject]
- [`KelletubeFullArtistObject`][KelletubeFullArtistObject]

## Pagination Response

`KelletubePaginationResponseObject` is generic model. The `items` property can contain any type of `Object` in it.

This is the structure of `KelletubePaginationResponseObject`:

| Property   | Type                                            |
| ---------- | ----------------------------------------------- |
| limit      | `number`                                        |
| nextOffset | `number` or `null`                              |
| total      | `number`                                        |
| hasMore    | `boolean`                                       |
| items      | List of generic type `T` which extends `Object` |
